Transaction Deduplication

Duplicate Payment Prevention — avoid double-charging on bank transfers using Redis SET with NX + EX flags.

How it works
  1. An application attempts to write data to a database.
  2. Deduplication check via hash stored in a Redis Set or probabilistic data type:
    a. If data does not exist in Redis → allow the write to the database.
    b. If data already exists → deny (duplicate detected).

Code Showcase

1. SETNX Pattern (SET NX EX)

// Atomic check-and-set: only succeeds if key doesn't exist
// SET workshop:dedup:{txHash} "1" NX EX 300

String txHash = sha256(sender + receiver + amount + timeWindow);
String key = "workshop:dedup:" + txHash;

// SET with NX (Not eXists) + EX (Expire) — atomic dedup!
Boolean wasSet = redis.opsForValue()
    .setIfAbsent(key, "1", Duration.ofSeconds(300));

if (Boolean.TRUE.equals(wasSet)) {
    // First time — process the payment
    return "ACCEPTED";
} else {
    // Key already exists — duplicate detected!
    return "DUPLICATE";
}

2. Redis CLI Equivalent

# First attempt — key created, payment processed
SET workshop:dedup:a1b2c3d4 "1" NX EX 300
# → OK

# Second attempt (within 5 min) — blocked!
SET workshop:dedup:a1b2c3d4 "1" NX EX 300
# → (nil)   ← duplicate detected

# Check remaining TTL
TTL workshop:dedup:a1b2c3d4
# → (integer) 287

# After TTL expires, same tx is accepted again
SET workshop:dedup:a1b2c3d4 "1" NX EX 300
# → OK

3. Bloom Filter Alternative (High Volume)

# For millions of tx/day, use a Bloom filter
# Probabilistic — tiny false-positive rate, zero false-negatives
# Much less memory than storing every key

# Create filter (1M capacity, 0.01% error rate)
BF.RESERVE workshop:dedup:bloom 0.0001 1000000

# Check + add in one step
BF.ADD workshop:dedup:bloom "a1b2c3d4e5f6"
# → (integer) 1  ← new item, not seen before

BF.ADD workshop:dedup:bloom "a1b2c3d4e5f6"
# → (integer) 0  ← already exists = DUPLICATE

# Check without adding
BF.EXISTS workshop:dedup:bloom "a1b2c3d4e5f6"
# → (integer) 1  ← exists
// Java equivalent with Lettuce + RedisBloom
// Note: Bloom filters don't support per-item TTL
// Combine with time-bucketed filter names:
//   "workshop:dedup:bloom:2024-01-15-14"  (hourly bucket)

// BF.ADD via Lettuce custom command
Long added = redis.execute((RedisCallback<Long>) conn ->
    conn.commands().execute("BF.ADD",
        "workshop:dedup:bloom".getBytes(),
        txHash.getBytes())
);

if (added == 1L) {
    return "ACCEPTED";   // new transaction
} else {
    return "DUPLICATE";  // already seen
}
