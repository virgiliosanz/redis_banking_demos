Authentication Token Store

How it works

  1. A user enters their credentials.
  2. An identity provider validates the credentials and generates an authentication token.
  3. The token is stored in Redis for the duration of the session. Any component can validate the user by checking Redis.

Live Demo

Scenario: Generate an auth token on login, store it in a Redis Hash with TTL, validate it, and destroy it on logout.

● VALID

Auth Token (Redis Hash)

TTL 300s
Token expired — TTL reached zero. Key removed from Redis.

Code Showcase

// 1. Generate token & store as Hash — HSET
String tokenId = UUID.randomUUID().toString();
String key = "workshop:auth:token:" + tokenId;

Map<String, String> tokenData = Map.of(
    "tokenId",  tokenId,
    "username", "user1",
    "fullName", "Ana García López",
    "role",     "Premium Client",
    "issuedAt", String.valueOf(System.currentTimeMillis())
);

// HSET — store all token fields atomically
redis.opsForHash().putAll(key, tokenData);

// EXPIRE — auto-delete after 300 seconds
redis.expire(key, 300, TimeUnit.SECONDS);

// 2. Validate token — HGETALL
Map<Object, Object> data =
    redis.opsForHash().entries(key);
// Empty map = token expired or invalid

// 3. Check remaining TTL — TTL command
Long ttl = redis.getExpire(key,
                           TimeUnit.SECONDS);

// 4. Logout — DEL (destroy token)
redis.delete(key);
# 1. Store auth token as Hash
HSET workshop:auth:token:a1b2c3d4-uuid \
  tokenId "a1b2c3d4-uuid" \
  username "user1" \
  fullName "Ana García López" \
  role "Premium Client" \
  issuedAt "1713045600000"

# 2. Set TTL (300 seconds = 5 min)
EXPIRE workshop:auth:token:a1b2c3d4-uuid 300

# 3. Validate — read all token fields
HGETALL workshop:auth:token:a1b2c3d4-uuid

# 4. Read single field
HGET workshop:auth:token:a1b2c3d4-uuid username

# 5. Check remaining TTL
TTL workshop:auth:token:a1b2c3d4-uuid

# 6. Logout — delete token
DEL workshop:auth:token:a1b2c3d4-uuid
