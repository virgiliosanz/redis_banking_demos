Distributed Locking

How it works
  1. Lock request: Client sends SET workshop:lock:{accountId} {clientId} NX EX 30 — atomic acquire with TTL.
  2. NX guarantee: Only the first client succeeds. Others get a denial — mutual exclusion enforced by Redis.
  3. Safe release: A Lua script checks that the caller owns the lock before deleting — prevents accidental release.
  4. Auto-expiry: EX ensures locks are released even if the holder crashes — no deadlocks.

Live Demo

Scenario: Lock a bank account during a wire transfer. Only one service instance can hold the lock — others are denied until it's released or expires.

// 1. Acquire lock — SET NX EX (atomic)
String key = "workshop:lock:" + accountId;
Boolean acquired = redis.opsForValue()
    .setIfAbsent(key, clientId,
        Duration.ofSeconds(30));  // NX + EX

if (acquired) {
    // Lock acquired — proceed with transfer
} else {
    // Lock held by another client — retry or fail
}

// 2. Safe release — Lua script
// Only delete if WE hold the lock
String luaScript =
    "if redis.call('GET', KEYS[1]) == ARGV[1] then " +
    "  return redis.call('DEL', KEYS[1]) " +
    "else " +
    "  return 0 " +
    "end";

Long result = redis.execute(
    new DefaultRedisScript<>(luaScript, Long.class),
    List.of(key),   // KEYS[1]
    clientId         // ARGV[1]
);
// result == 1 → released, 0 → not our lock

// 3. Check lock status
String holder = redis.opsForValue().get(key);
Long ttl = redis.getExpire(key, TimeUnit.SECONDS);

                    


                    

                        
# 1. Acquire lock (atomic SET NX EX)
SET workshop:lock:acc-1001 transfer-svc-1 NX EX 30
# → OK if acquired, (nil) if already held

# 2. Check who holds the lock
GET workshop:lock:acc-1001
# → "transfer-svc-1"

# 3. Check remaining TTL
TTL workshop:lock:acc-1001
# → 28 (seconds remaining)

# 4. Try to acquire from another client (DENIED)
SET workshop:lock:acc-1001 transfer-svc-2 NX EX 30
# → (nil)  ← NX prevents overwrite

# 5. Safe release with Lua script
EVAL "if redis.call('GET', KEYS[1]) == ARGV[1] \
  then return redis.call('DEL', KEYS[1]) \
  else return 0 end" \
  1 workshop:lock:acc-1001 transfer-svc-1
# → 1 (released)

# 6. Wrong client tries to release (DENIED)
EVAL "if redis.call('GET', KEYS[1]) == ARGV[1] \
  then return redis.call('DEL', KEYS[1]) \
  else return 0 end" \
  1 workshop:lock:acc-1001 transfer-svc-2
# → 0 (not your lock!)
