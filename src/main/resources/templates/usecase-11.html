Real-time Transaction Monitoring

Live transaction metrics with Redis Time Series — TS.CREATE, TS.ADD, TS.RANGE with aggregation.

How it works

  1. Ingest: Each transaction generates data points — count, amount, risk score — written to Redis Time Series with TS.ADD.
  2. Store: Redis Time Series stores timestamped data with automatic retention (1 hour) and duplicate policies (SUM for counts, LAST for values).
  3. Query: TS.RANGE with AGGREGATION retrieves data in time buckets (1-second windows) for real-time dashboards.
  4. Detect: Anomaly bursts (spike in volume + high amounts) are instantly visible in the time series data.

Code Showcase

# ═══ Redis Time Series — Transaction Monitoring ═══

# Create time series with 1-hour retention
TS.CREATE workshop:txmonitor:ts:count \
  RETENTION 3600000 DUPLICATE_POLICY SUM

TS.CREATE workshop:txmonitor:ts:amount \
  RETENTION 3600000 DUPLICATE_POLICY LAST

# Add data points (timestamp = * means auto)
TS.ADD workshop:txmonitor:ts:count * 1
TS.ADD workshop:txmonitor:ts:amount * 250.00
TS.ADD workshop:txmonitor:ts:risk_score * 35.0

# Query last 60 seconds in 1-second buckets
TS.RANGE workshop:txmonitor:ts:count \
  - + AGGREGATION SUM 1000

# Average amount per 5-second bucket
TS.RANGE workshop:txmonitor:ts:amount \
  - + AGGREGATION AVG 5000

# Max risk score per 10-second bucket
TS.RANGE workshop:txmonitor:ts:risk_score \
  - + AGGREGATION MAX 10000
// ═══ Redis Time Series with RedisCallback ═══
// Spring Data Redis doesn't have TS.* templates,
// so we use execute() for raw commands

// Create a time series
redis.execute((RedisCallback<Object>) connection ->
    connection.execute("TS.CREATE",
        "workshop:txmonitor:ts:count".getBytes(),
        "RETENTION".getBytes(), "3600000".getBytes(),
        "DUPLICATE_POLICY".getBytes(), "SUM".getBytes()));

// Add a data point
long timestamp = System.currentTimeMillis();
redis.execute((RedisCallback<Object>) connection ->
    connection.execute("TS.ADD",
        "workshop:txmonitor:ts:count".getBytes(),
        String.valueOf(timestamp).getBytes(),
        "1".getBytes()));

// Query with aggregation (SUM per 1-second bucket)
long now = System.currentTimeMillis();
long from = now - 60000; // last 60 seconds
