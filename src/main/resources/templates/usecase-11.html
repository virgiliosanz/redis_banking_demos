Real-time Transaction Monitoring

Live transaction metrics with Redis Streams — XADD, XRANGE, XLEN with manual aggregation.

How it works

  1. Ingest: Each transaction is written to a Redis Stream with XADD — amount and risk score as fields.
  2. Store: Redis Streams store timestamped entries with automatic IDs (ms-precision). XTRIM MAXLEN keeps the stream bounded.
  3. Query: XRANGE reads entries in a time window. The app aggregates into 1-second buckets (COUNT, AVG, SUM) for the dashboard.
  4. Detect: Anomaly bursts (spike in volume + high amounts) are instantly visible in the aggregated stream data.

# ═══ Redis Streams — Transaction Monitoring ═══

# Add transaction events (auto-generated timestamp ID)
XADD workshop:txmonitor:stream * \
  amount 250.00 riskScore 35.0

XADD workshop:txmonitor:stream * \
  amount 15000.00 riskScore 82.5

# Query last 60 seconds (IDs are ms-based)
# From = now - 60000, To = now
XRANGE workshop:txmonitor:stream 1713045540000 1713045600000

# Stream length (total entries)
XLEN workshop:txmonitor:stream

# Trim to keep last 10000 entries
XTRIM workshop:txmonitor:stream MAXLEN ~ 10000

# Read most recent 5 entries
XREVRANGE workshop:txmonitor:stream + - COUNT 5
